## 多角度定义 Agent 自进化

请从多个角度定义 Agent 自进化，而不是只给一个口号式定义。

至少包括：

### 1. 从 Prompt 优化角度

Agent 自进化可以表现为系统根据历史任务表现自动改写、压缩、增强 Prompt。

### 2. 从 Skill 沉淀角度

Agent 自进化可以表现为系统把成功经验、失败经验、操作流程沉淀成可复用 Skill。

### 3. 从 Memory 角度

Agent 自进化可以表现为系统把长期偏好、上下文、经验记录进长期记忆，并在后续任务中调用。

### 4. 从 Reflection 角度

Agent 自进化可以表现为 Agent 在失败后进行自我反思、错误定位和策略修正。

### 5. 从 Trace 角度

Agent 自进化可以表现为系统记录 Agent 的完整执行轨迹，包括规划、工具调用、错误、反馈和最终结果，并从中提取可复用经验。

### 6. 从后训练角度

Agent 自进化可以表现为系统将高质量执行轨迹转化为训练数据，通过 SFT、DPO、RLAIF、RL 等方式更新模型或 Adapter。

### 7. 从工程系统角度

Agent 自进化可以表现为一个持续闭环：

```text
Agent 执行
→ Trace 记录
→ 数据清洗与筛选
→ 样本构造
→ 后训练
→ 自动评测
→ 灰度发布
→ 线上反馈
→ 下一轮迭代
```

最后请给出一个你认为最适合工程实践的综合定义。

## 技术层次梳理

请将 Agent 自进化拆成由浅到深的技术层次。

建议层次如下：

1. **Prompt 自优化**
   - 解决什么问题；
   - 优点是什么；
   - 局限是什么。

2. **Skill / Workflow 沉淀**
   - 如何把经验固化为可复用流程；
   - 为什么它比单纯 Prompt 更稳定；
   - 为什么它还不等于模型能力提升。

3. **Memory / Long-term Context**
   - 记忆系统如何支持个性化和长期任务；
   - 记忆污染、检索错误和上下文膨胀的问题是什么。

4. **Reflection / Self-Correction**
   - 自反思为什么能提升复杂任务表现；
   - 反思的成本、循环和错误强化风险是什么。

5. **Trace / Agent Behavior Data**
   - 为什么执行轨迹是重要数据资产；
   - Trace 里哪些内容可能有训练价值；
   - 如何区分成功轨迹、失败轨迹和修复轨迹。

6. **Evaluation / Regression**
   - 没有评测，自进化无法闭环；
   - 为什么自动评测、人工评测、线上评测都需要。

7. **Trace-to-Dataset**
   - 如何从 Agent 执行轨迹构造 SFT、DPO、Reflection、Tool-use 训练样本；
   - 数据质量门禁为什么重要。

8. **Post-training / Agentic RL**
   - SFT、DPO、RLAIF、RL 分别适合优化什么；
   - 为什么短期工程落地不应一上来追求复杂 RL；
   - Agentic RL 的长期潜力是什么。

9. **Deployment Feedback Loop**
   - 模型注册、灰度发布、回滚、监控如何组成工程闭环；
   - 为什么“自动训练”不等于“自动上线”。
