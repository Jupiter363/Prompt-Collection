# 旅行规划助手技术方案生成提示词（优化版）

## 优化后的提示词

你是一名资深 **AI Agent 工程师 / LangChain & LangGraph 实践导师 / 技术方案架构师**。  
我目前是一个 **Agent Python 技术栈初学者**，对 LangChain、LangGraph 等 Python Agent 框架还不太熟悉，只理解一些基础思想，例如：Agent 可以调用工具、可以按照流程执行任务、可以根据中间结果反思修正，但缺少完整的工程实践经验。

现在我希望通过一个小型实践项目来进一步学习 Agent 工程能力。请你帮我设计一个 **旅行规划助手 Travel Planning Agent** 的技术方案，并最终输出为 **Markdown 格式的技术文档**。

---

## 一、项目目标

我希望搭建一个面向学习和实践的旅行规划助手，它不需要一开始就做得非常复杂，但需要体现主流 Agent 工程思想，帮助我理解以下能力：

1. 如何用 Python 搭建一个基础 Agent 应用；
2. 如何使用 LangChain 完成 LLM、Prompt、Tool、Chain 的基础封装；
3. 如何使用 LangGraph 编排 Agent 工作流；
4. 如何设计 Workflow，让任务按明确流程推进；
5. 如何加入 Reflection 反思机制，让 Agent 对初步旅行方案进行自检和优化；
6. 如何接入工具调用，例如联网搜索、地点信息查询、天气查询、预算估算等；
7. 如何将一个模糊的旅行需求逐步转化为结构化旅行计划。

---

## 二、项目场景

请以“旅行规划助手”为核心场景进行设计。

用户输入示例：

> 我想 6 月底去成都玩 3 天，预算 3000 元，喜欢美食和轻松一点的路线，请帮我规划。

系统需要完成的事情包括：

1. 理解用户的旅行需求；
2. 提取目的地、时间、天数、预算、偏好、同行人等关键信息；
3. 判断信息是否足够，不足时提出补充问题；
4. 必要时调用工具获取外部信息，例如：
   - 景点信息；
   - 美食推荐；
   - 天气情况；
   - 交通建议；
   - 酒店区域建议；
   - 最新营业时间或门票信息；
5. 生成初版旅行计划；
6. 使用 Reflection 机制检查计划是否合理；
7. 根据反思结果优化计划；
8. 输出最终旅行方案。

## 三、技术要求

请围绕以下技术栈设计方案：

### 1. Python

项目以 Python 为主要开发语言。

### 2. LangChain

需要说明 LangChain 在项目中的作用，例如：

- PromptTemplate；
- ChatModel / LLM 调用；
- Tool 封装；
- Output Parser；
- Chain 组合；
- 工具调用接口封装。

### 3. LangGraph

需要说明 LangGraph 在项目中的作用，例如：

- StateGraph；
- State 状态设计；
- Node 节点设计；
- Edge / Conditional Edge 路由；
- START / END；
- 工作流编排；
- 多节点协作。

### 4. Workflow

请设计一个清晰的旅行规划工作流，例如：

1. 用户需求解析；
2. 信息完整性检查；
3. 工具调用补充信息；
4. 初版计划生成；
5. Reflection 反思检查；
6. 计划修正；
7. 最终结果输出。

### 5. Reflection

请设计一个简单但有效的反思机制，例如：

- 检查预算是否合理；
- 检查行程是否过于紧凑；
- 检查路线是否顺路；
- 检查是否符合用户偏好；
- 检查是否存在信息缺失；
- 如果发现问题，返回修改建议，并重新生成优化版计划。

### 6. 工具调用

请设计工具调用方案。工具可以包括但不限于：

- 联网搜索工具；
- 天气查询工具；
- 景点搜索工具；
- 地图/路线估算工具；
- 预算估算工具；
- 本地 mock 工具。

如果需要联网搜索，请说明：

- 初学阶段如何先用 mock 工具替代真实 API；
- 后续如何切换成真实搜索 API；
- 工具输入输出应该如何设计；
- 如何在 LangChain / LangGraph 中注册和调用工具。
